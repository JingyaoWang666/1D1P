# 1D1P — One Day One Problem

A personal knowledge base built with GitHub Issues, inspired by the **Feynman Technique**: if you can't explain it simply, you don't understand it well enough.

## 📖 Purpose

This repository is my **Daily Question** journal. Each day I turn something I learned — or something I'm curious about — into a concrete question and then answer it in my own words. The goal is to:

- Solidify new knowledge by explaining it clearly
- Build a searchable personal knowledge base over time
- Practice thinking from first principles
- Track intellectual growth, one question at a time

Questions can come from anything: a network concept I read about, a life observation, a piece of code that puzzled me, or a random "why does this work?" moment.

## 🗂️ How I Use GitHub Issues

Every daily entry is a **GitHub Issue** created from the [Daily Question template](.github/ISSUE_TEMPLATE/daily_question.md). Each issue:

- Is labeled **`daily-question`** for easy filtering
- Can be tagged with topic labels (e.g., `#network`, `#life`, `#coding`)
- Captures the date, the question, the thought process, and optional extensions
- Stays open until I'm satisfied with my answer, then gets closed

Browse all entries → [Issues tab](../../issues?label=daily-question)

## 🚀 Getting Started (for myself)

1. Go to **Issues → New Issue**
2. Choose the **Daily Question** template
3. Fill in the sections honestly — especially the *Thought Process* part
4. Add relevant labels and submit
